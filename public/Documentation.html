API Documentation

Toy API

HTTP Request Route Description Parameters
GET /toys Returns a list of toys according to the query parameters.
  • limit (optional): Number of items per page.
  • page (optional): Page number to return.
  • sort (optional): Sorting criteria.
  • s (optional): Search string for name or info fields.
GET /toys/category?cat= Returns a list of toys by category.
  • cat: Category name.
GET /toys/count Returns the total count of toys.
  • limit (optional): Number of items per page.
GET /toys/price Returns toys within a price range.
  • min (optional): Minimum price.
  • max: (optional) Maximum price.
GET /toys/single/:id Returns a single toy by its ID.
  • id: Toy ID.
POST /toys Create a new toy.
Token is required!
  • Request body toy details to be added in the following format.
  • name:String,
    info:String,
    category:String,
    img_url:String (allowed but not required),
    price:Number
PUT /toys/:id Update an existing toy.
Token is required!
  • id: Toy ID.
  • Request body toy details to be added in the following format.
  • name:String,
    info:String,
    category:String,
    img_url:String (allowed but not required),
    price:Number
DELETE /toys/:id Delete a toy by its ID.
Token is required!
  • id: Toy ID.
